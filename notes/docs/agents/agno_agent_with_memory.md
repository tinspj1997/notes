# Agno Agent with Memory

## Overview

This page documents an example Agno-based chat agent with Redis-backed memory. The agent retrieves contextual information from a knowledge base, uses a model compatible with OpenAI-like APIs, and persists conversational memory to Redis for later retrieval.

## Features

- Persistent conversational memory (Redis) with configurable expiry
- Uses an OpenAI-like model via `agno.models.openai.like.OpenAILike`
- Retrieves context using `fetch_context` tool
- Adds history and timestamps to context for grounded responses
- If no relevant context is found, agent responds with: "I don't have enough information to answer that question."

## Code

```python
from agno.agent import Agent
from agno.models.openai.like import OpenAILike


from src.core.artifacts.tools.chat import fetch_context
from src.core.utils.prompt import read_prompt
from agno.db.redis import RedisDb

# Initialize Redis with memory retention time from Default config (3 days = 259,200 seconds)
redis_db = RedisDb(
    db_url=Default.REDIS_URL,
    db_prefix="agent",
    expire=int(Default.CHAT_MEMORY_EXPIRY),  # Memory retention in seconds
)

model = OpenAILike(
    id="chat-agent-model",
    api_key="key",
    base_url="endpoint",
)

# - If the context does not contain relevant information, respond with "I don't have enough information to answer that question."
ChatAgent = Agent(
    name="ChatAgent",
    description="Handles conversational user queries by retrieving relevant context from the knowledge base and delivering accurate, grounded responses.",
    model=model,
    instructions=read_prompt("chat"),
    add_datetime_to_context=True,
    markdown=True,
    tools=[fetch_context],
    db=redis_db,
    update_memory_on_run=True,
    add_history_to_context=True,
    num_history_runs=20,
)
```

## Explanation

- Redis memory: The `RedisDb` instance persists conversational memory with a prefix of `agent` and expiry controlled by `Default.CHAT_MEMORY_EXPIRY` (default shown as 3 days = 259,200 seconds).
- Model: `OpenAILike` is configured using the project's `Default` constants for model name, API key, and endpoint.
- Tools: `fetch_context` is provided so the agent can query the knowledge base and ground responses.
- Behavior: The agent is configured to update memory on each run, include recent history in context (up to `num_history_runs` entries), and append timestamps to context items.
- Fallback: If the retrieved context does not contain relevant information, the agent should return the explicit fallback message.

## Notes & Integration

- Ensure environment variables or `Default` constants are set: `REDIS_URL`, `CORE42_API_KEY`, `CORE42_API_ENDPOINT`, and `QUERY_MODEL_NAME`.
- Tune `CHAT_MEMORY_EXPIRY` according to retention policy and storage costs.
- This example expects `agno` and the project's `src.core` modules to be importable from your runtime.

## Example usage

Import and run `ChatAgent` within your application, passing user input through the agent's run/execute interface (per your project's Agno conventions). The agent will retrieve context via `fetch_context`, consult memory in Redis, and produce a grounded response or the fallback when no relevant context is available.
